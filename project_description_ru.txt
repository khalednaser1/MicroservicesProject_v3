Описание проекта: Microservices API Gateway

Разработан API Gateway (Backend for Frontend — BFF), который объединяет данные из нескольких микросервисов и предоставляет клиентам единое REST API. Сервис поддерживает агрегацию ответов, кэширование данных и маршрутизацию запросов.

Цель проекта:
Спроектировать слой Gateway между клиентами и микросервисами, объединять данные из разных источников, применять кэширование, аутентификацию и обработку ошибок.

Компоненты системы:
✅ **User Service** (порт 6003) — хранит информацию о пользователях.
✅ **Product Service** (порт 6001) — хранит данные о товарах.
✅ **Order Service** (порт 6002) — возвращает список заказов пользователя.
✅ **API Gateway** (порт 8080) — агрегирует данные и предоставляет клиентам единый REST API.
✅ **Redis** (порт 6379) — кэширование данных.

Реализованный сценарий:
**GET /api/profile/{userId}**
Gateway объединяет данные из трёх сервисов и возвращает общий JSON-ответ, кэшируя результат на 30 секунд.

Выполненные функциональные требования:
✅ **REST API с агрегацией данных** — единая точка доступа /api/profile/{id}
✅ **Кэширование Redis** — TTL 30 секунд для агрегированных ответов
✅ **Retry и fallback** — Polly Retry Policies (3 попытки, экспоненциальная задержка)
✅ **Авторизация JWT** — endpoint /token для получения access_token
✅ **Health Checks** — endpoint /health для мониторинга состояния

Выполненные нефункциональные требования:
✅ **Логирование:** Serilog (структурированное логирование в консоль)
